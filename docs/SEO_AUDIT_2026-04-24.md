# SEO аудит TechnoRent — 24.04.2026

## Що перевірено

- Базові meta-теги публічних сторінок.
- Canonical URL для головної, каталогу, послуг, контактів, деталей техніки і деталей послуг.
- Open Graph і Twitter Card для поширення сторінок у месенджерах.
- JSON-LD для головної, техніки, послуг і FAQ.
- `robots.txt`.
- `sitemap.xml`.
- Дублювання legacy-сторінки `/vyviz-smittia` з актуальною сторінкою послуги.
- Production build клієнта і сервера.

## Що виправлено

- Додано підтримку `og:image` і Twitter Card у `PageMeta`.
- Додано спільне fallback-зображення для SEO preview основних спискових сторінок.
- Для деталей техніки і послуг зображення для SEO приводяться до абсолютного URL.
- Для сторінок каталогу, послуг і контактів додано/посилено OG/Twitter preview.
- Для сторінки контактів додано повні OG/Twitter meta.
- Для legacy-лендінгу `/vyviz-smittia` canonical і JSON-LD URL переведено на `/services/vyviz-budivelnogo-smittia`.
- Динамічний sitemap тепер доступний за двома адресами:
  - `/sitemap.xml`;
  - `/api/sitemap.xml` як сумісний старий alias.
- Із sitemap прибрано legacy `/vyviz-smittia`, щоб не створювати дубль сторінки послуги.
- `robots.txt` тепер вказує на `/sitemap.xml` і закриває від індексації:
  - `/admin`;
  - `/api/admin/`;
  - `/api/internal/`.
- Статичний `client/public/sitemap.xml` оновлено як fallback для головних статичних сторінок.

## Поточна поведінка

- Основний sitemap генерується на backend з БД і включає:
  - `/`;
  - `/catalog`;
  - `/services`;
  - `/contacts`;
  - всі активні послуги;
  - всю техніку.
- `lastmod` для техніки і послуг береться з `updatedAt`.
- Sitemap кешується браузерами/ботами на 1 годину.
- `robots.txt` також кешується на 1 годину.

## Перевірка

Виконано:

```bash
npm run build:server
npm run build:client
```

Результат: обидві збірки проходять. Vite показує лише попередження про великий JS chunk, це не блокує деплой.

## Що лишилось на майбутнє

- Якщо production-домен лишається `technorent.lanbox.com.ua`, потрібно узгодити `SITE_URL` і canonical URL. Зараз SEO-ціль у коді — `https://technorent.ua`.
- Додати реальні контактні телефон/email замість заглушок.
- Замінити зовнішні fallback-зображення на власні production-фото.
- Розбити великий frontend chunk через lazy routes, якщо буде задача покращити перше завантаження.
- Після деплою перевірити `/sitemap.xml`, `/robots.txt` і кілька сторінок через Google Search Console або аналогічний валідатор.
