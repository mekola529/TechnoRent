# План: Публічна інформація замовлення в кабінеті клієнта

## Вже впроваджено в цьому етапі

1. Customer API віддає нормалізовані поля виконання:
   - `executionAddressFrom`
   - `executionAddressTo`
   - `executionScheduledDate`
   - `executionScheduledTime`
   - `statusLabel`
   - `rawStatus`
2. Якщо заявка переведена в замовлення, дані з `RentOrder` мають пріоритет над початковою заявкою.
3. Якщо це евакуатор, клієнт бачить дві адреси: адресу подачі та адресу доставки.
4. Статус для клієнта відділений від технічного статусу в базі.

## Публічні статуси для клієнта

- `Очікує підтвердження менеджером`
- `Підтверджено`
- `Виконується`
- `Очікує оплати`
- `Завершено`

## Наступний етап: контакти працівника

1. Додати в `RentOrder` колонку:

```sql
ALTER TABLE "RentOrder"
ADD COLUMN IF NOT EXISTS "showWorkerToCustomer" BOOLEAN NOT NULL DEFAULT false;
```

2. В адмінці в деталях замовлення додати checkbox:

```text
Показувати працівника клієнту
```

3. Customer API має віддавати контакти працівника тільки якщо:
   - `RentOrder.showWorkerToCustomer = true`;
   - є активне/прийняте призначення в `WorkAssignment`;
   - працівник не видалений/не заблокований.

4. Дані для клієнта:

```ts
workerContact: {
  fullName: string;
  phone: string | null;
} | null
```

5. У кабінеті клієнта блок “Працівник”:
   - якщо `workerContact` є, показати імʼя та телефон;
   - якщо немає, показати нейтральний текст без технічних деталей.

## Важливе правило безпеки

Контакти працівника не можна віддавати на frontend, якщо менеджер не увімкнув прапорець. Приховування тільки через CSS або UI недостатнє.
